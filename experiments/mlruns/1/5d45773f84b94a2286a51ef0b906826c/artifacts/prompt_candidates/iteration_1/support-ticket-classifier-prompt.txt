Instructions for Classifying Customer Support Messages:

Task Description:
You are tasked with classifying customer support messages into one of four distinct categories. Each message you receive will contain information regarding customer inquiries, issues, or requests. Your goal is to accurately classify each message into one of the defined categories. It is crucial that you provide exactly one category name as the output, without any additional explanations or text.

Categories:
- Incident: This refers to any unexpected issue that demands immediate attention and action. Look for messages that describe urgent, unforeseen problems that need to be resolved swiftly to prevent further complications.

- Request: This category focuses on routine inquiries or service requests. These are typically straightforward communications where the customer is asking for information, clarification, or standard assistance that doesn't involve critical problems.

- Problem: Here, the focus is on underlying or systemic issues that result in multiple incidents. These messages might highlight ongoing, unresolved issues that indicate a larger problem within the system or service.

- Change: Messages in this category involve planned changes, updates, or configuration requests. These are deliberate actions intended to alter or improve existing systems, services, or configurations as a part of standard operations or strategic development.

Classification Strategy:
1. Carefully read the customer message to identify the main theme and intent.
2. Determine whether the issue is urgent and unexpected (Incident), or if it is a standard request for information or service (Request).
3. Check if the message describes a larger, underlying issue that suggests systemic troubles (Problem).
4. Look for indications of a planned or requested change in systems, configurations, or services (Change).
5. Select the category name that best fits the content and nature of the customer message.
6. Output only the category name with no additional explanation.

Additional Notes:
- Avoid making assumptions beyond the content of the message.
- Maintain accuracy by aligning your choice with the definitions provided for each category.
- Consistency is key; apply the same logic and criteria across all messages to ensure reliable classification results.
